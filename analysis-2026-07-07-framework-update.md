## 二、正反論述裁決（Threads 貼文 vs 財務工程批判文）

**核心裁決：兩篇回答的是不同問題，可同時成立。**

| | 第一篇（MSTR Monday） | 第二篇（財務工程批判） |
| --- | --- | --- |
| 回答的問題 | 未來 6 個月價格怎麼走 | 這個結構長期能不能成立 |
| 最強論點 | 流量論：$216M 對 BTC 日成交量是雜訊，訊號 > 流量 | Tranche 類比：優先股=senior、普通股=equity、Monetization=triggered liquidation，pro-cyclical 被改為 **delayed pro-cyclical** → 見 [[delayed-pro-cyclical]] |
| 最大盲點 | 「一定是淨買家」——淨買家是 **mNAV>1 的函數**，不是 Saylor 意志的函數 | 「ATM 撐儲備」已過時：6/29–7/5 零 ATM、改賣幣；「人為創造溢價」是舊 regime 的質疑 |
| 需要校正 | BTC 95k→$570 / 150k→$1000 是把四年週期線性外推，而週期指標本輪已集體失效（見第四節） | 折價期公司的套利方向反轉：$1B 回購=公司自己當 AP 做折價套利，這個方向對股東增值 |

**量化補充（慢性失血率）**：USD 儲備 $2.55B = 17.4 個月覆蓋 → 年化優先股股息+利息義務 ≈ **$1.76B**。對 ~$50B BTC NAV 而言 = **每年 ~3.5% 的自然損耗率**。結論：不是死亡螺旋，是慢性失血；BTC 橫盤時間是主要變數。

**「兩個方向」的實務含義（第二篇最隱晦處）**：
- mNAV > 1（舊 regime）：公司動機 = 撐高溢價 → 增發買幣飛輪。
- mNAV < 1（現況）：公司握 $1B 普通股回購額度，**買得越便宜對留存股東越增值** → 回購執行期間，公司未必希望股價快速回升。
- ⚠️ 對持有短天期多頭部位（如年底 OTM call）的直接含義：除 BTC 價格風險外，多一層「公司回購動機與你的時間對賭」的逆風。LEAP 受影響較小，現股不受影響。

## 三、兩等份觸發框架修訂提案

### 提案 A：第 2 等份條件③ 重新定義「無被迫賣幣」

原條件：`MSTR 無被迫賣幣且 mNAV ≥ 1.0x`

問題：Digital Credit Capital Framework 生效後，**每週賣幣已是設計內行為**，「有沒有賣」不再是有效的二元訊號。有效訊號是「賣的量級」。

修訂為：

```
③ 賣幣節奏 ≤ 營運需求 且 mNAV ≥ 1.0x
   營運需求基準：股息+利息年化 ~$1.76B ≈ $147M/月 ≈ $34M/週
   - 週賣幣 ≤ ~1.5x 基準（~$50M）：正常營運，✅
   - 週賣幣 > 2x 基準 或 開始動用 $1.25B 額度：⚠️ 性質改變（付帳單 → 主動減倉）
   - 連續兩週 > 2x 基準：❌ 視同「被迫賣幣」觸發
```

（本週 $216M 遠超週基準，但屬四天連假後補繳＋一次性補儲備，建議標記為⚠️觀察一次、下週回歸基準判斷。）

### 提案 B：新增每日監控欄位

| 新欄位 | 理由 | 訊號解讀 |
| --- | --- | --- |
| **STRC 價格 vs $100 面額** | 官方目標區間 $99–100；它是整個優先股體系的信任投票 | < $95 = 普通股仍在替 senior tranche 墊背；回到 $99+ 前，mNAV 難以重估 |
| **$1.25B 額度使用率** | 目前 0%；使用速度 = 壓力表 | 首次動用即⚠️；使用率 > 50% 即❌ |
| **$1B 回購執行進度（普通股/優先股分列）** | 折價期的公司行為方向盤 | 積極回購+股價壓抑 = 符合「兩個方向」推論，短天期多單逆風確認 |
| **週賣幣金額 / 週股息需求 比值** | 提案 A 的輸入 | > 2 連續兩週 = 觸發 |

### 提案 C：mNAV regime 標籤

在每日快照的 `MSTR mNAV` 欄加註 regime：
- `mNAV ≥ 1.0`：**增發 regime**（舊飛輪可運轉，[[mnav-reflexivity]] 正向）
- `0.8–1.0`：過渡帶
- `< 0.8`：**回購 regime**（公司=折價套利者，與小股東短期利益存在張力）

目前：0.68x = 回購 regime。第 2 等份要求 mNAV ≥ 1.0x 的深層意義即「確認 regime 已切換回增發模式」，維持不變。

## 四、週期指標失效：對「③週期遞減」維度的衝擊

### Pi Cycle Top：本輪正式失效（首次）

- 2025/10 BTC 見頂 $126k，111DMA **從未**上穿 2×350DMA——兩線反覆靠近又分開。
- 且非單一失效：MVRV（頂部僅 3.8–4.2，不到歷史警戒線一半）、NUPL、Puell、Reserve Risk、RHODL **集體**未進危險區。
- 結構性原因：Pi Cycle Top 量測的是**上漲角度**，需要散戶 FOMO 式拋物線；本輪邊際買家是按季配置的機構，$40k→$126k 走了 20 個月，角度完全不同。MVRV 失真同理：ETF/Strategy 的幣躺在託管錢包從未鏈上移動，realized cap 被扭曲。
- **結論：這是 regime change，不是雜訊。** 詳見 [[indicator-regime-change]]。

### Pi Cycle Bottom：建議降級為「加分項」

- 現狀：150EMA 尚未下穿 471SMA×0.745（6 月中確認未觸發）。
- 三個不信任理由：
  1. **Curve fitting，n≈2**：參數（471/150/0.745）是對 2015、2018 兩個底反覆擬合湊出來的，有效樣本比 Top 更少。
  2. **殺死 Top 的原因對 Bottom 同樣適用**：舊底部由散戶槓桿連環爆倉的 −80% 急殺形成；若本輪機構市場跌得「慢而淺」（−50~55% 後長橫盤），EMA 可能永遠追不上交叉條件——在場外等一個永遠不來的訊號。
  3. 即使觸發，它辨識的是舊 regime 的底部形態，資訊含量已折損。

### 對現有框架的具體影響

1. **「歷史底部窗口 12–14 月」需加折扣**：此窗口源自四年週期，而本輪頂部已證明週期類指標集體失準。建議：③週期遞減維度**權重下調**或窗口放寬為 12–18 月，並在計分卡註明「四年週期前提已弱化」。
2. **Benchmark $570（BTC 95k）目標價的前提**即「四年週期十月重啟」——用剛失效的框架推年底牛市，引用時應打折。分析師離散度本身就是訊號：Wells Fargo $54 ↔ Benchmark $570，共識已壓縮至低 $300 區。
3. **底部判斷改用一籃子獨立證據**（維持現行五維但調整權重）：LTH 行為、ETF 流出枯竭度（6 月 −$4B 流出歷史上偏底部反指標）、200WMA、宏觀流動性；Pi Cycle Bottom 觸發=加分項，**不觸發≠底沒到**。
4. MVRV-Z ≤ 0（第 1 等份②）同樣是舊 regime 校準的門檻：託管化使 realized cap 失真，MVRV-Z 可能在真實底部仍 > 0。建議：②改為「MVRV-Z ≤ 0 **或** 6 個月滾動 Z-score 進入最低 5% 分位」雙軌。
