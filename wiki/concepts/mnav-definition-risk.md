# mnav-definition-risk（mNAV 定義權風險）

**一句話**：mNAV 既是 Strategy 公開的決策變數（發行指引以它分層），其定義權又在公司自己手上（strategy.com/notes 自訂），且數值可透過發行特別股機械式移動——裁判、球員、計分板同一人。

## 機械式墊高的算術

官方 mNAV = EV ÷ BTC NAV，EV 中特別股按**清算面額**計：

```
情境：mNAV < 1，STRC 市價 $80（面額 $100）
發行 1 單位 STRC：
  分子 +$100（面額）
  分母 +$80（實收現金買入 BTC）
→ 折價越深，墊高效果越強
→ 即使平價發行，分子分母同加也使 <1 的比值向 1 收斂
```

⚠️ 公司最需要 mNAV「看起來 ≥1」的折價期，恰好是這個工具效果最強的時期。

## 常見誤解的校正

「公司叫你看 mNAV、自己看 GAAP NAV」——**不成立**：
1. 2025/8 的 8-K 發行指引整套以 mNAV 分層（<2.5x 戰術性 / 2.5–4x 機會性 / >4x 積極 / <1x 考慮發債回購），最新框架也明示在 ~1x mNAV 時對普通股發行保持紀律。
2. ASU 2023-08 後 BTC 按公允價值入帳，GAAP NAV ≈ BTC NAV，這條軸區分不出東西。

**修正後的結論更嚴重**：決策變數 = mNAV → 定義權 = 公司自訂 → 數值 = 公司可移動。外加歷史前科：2.5x 指引附帶「當公司認為有利時」的裁量逃生門，且已被使用過。

## 防污染四原則（本 repo 採用）

1. **自算不引用**：官網 mNAV 降級為情緒參考，觸發輸入用自算值（見 `metrics-spec.md` M1/M2）。
2. **雙軌取保守**：equity mNAV 與 enterprise mNAV 同時達標才算數。
3. **特別股稀釋旗標**：mNAV 回升伴隨特別股淨增發時，訊號打五折（M3）。
4. **不可操縱交叉確認**：STRC 市價、覆蓋月數、sats/股——定義權不在公司手上的三個錨。
