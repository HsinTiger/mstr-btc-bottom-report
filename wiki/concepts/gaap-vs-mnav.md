# GAAP vs Non-GAAP vs mNAV

**一句話**：GAAP 是法定財報語言，Non-GAAP 是公司自訂溝通語言；mNAV/BTC Yield/BTC $ Gain 可以有資訊價值，但定義權在公司手上，不能當成查核過的普通股安全邊際。

## 三條公式不要混用

### 1. BTC NAV

```
BTC NAV = BTC 持倉 × BTC 現貨價
```

這只是 Strategy 資產負債表裡的「BTC 資產線」。它不含現金、不扣債、不扣稅，也不處理優先股。

### 2. GAAP NAV / 股東權益

```
GAAP NAV = BTC 公允價值 + 現金 + 其他資產 − 可轉債 − 淨遞延稅負債 − 其他負債
```

ASU 2023-08 讓 crypto assets 以公允價值入帳，但它只改變 BTC 那條資產線，沒有把整張資產負債表簡化成 BTC NAV。牛市中，未實現收益對應的遞延稅負債可能成為普通股淨值的重要楔子。

### 3. mNAV / BTC Yield / BTC $ Gain

這些是公司自訂的 Non-GAAP KPI。它們有助理解財庫反身性與每股含幣量敘事，但不是會計師查核的 GAAP 數字。

## 對 MSTR 持有分析的含義

1. **普通股不是 senior tranche**：優先股若在會計上列入權益，也不等於屬於普通股；估值要扣清算面額或至少明確做壓力測試。
2. **mNAV 是行為變數，不是安全邊際本身**：公開發行/回購溝通常掛 mNAV 分層，但現金流、股息覆蓋、債務條款與稅務才是硬約束。
3. **小倉合約要更保守**：MSTR 2.5x 波段只吃右側確認，不在 mNAV 定義混濁、STRC 折價或賣幣紅燈時賭反身性。
4. **大倉現貨看 1–4 年週期**：MSTR/BMNR 4:1 可容忍波動，但要分開看 BTC/ETH beta、每股含幣量、稀釋與資本結構品質。

## 對 BMNR 的類比

BMNR 不能只用「ETH 版 MSTR」估值。它需要獨立回答：

- ETH NAV 是否可驗證？
- 每股 ETH 是否真的增厚，還是總持幣增加但股本更快稀釋？
- 若有 BMNP/優先股或其他融資工具，普通股是否被放到更後順位？
- 市場給的 NAV premium 是流動性溢價、敘事溢價，還是可持續的融資飛輪？

因此 BMNR 在本體系中維持 20% 衛星上限，不升主倉，除非其每股 ETH 增厚效率與資本結構品質被證明。

## 操作規則

- 看到公司自創 KPI，先問：`10-Q 找得到嗎？定義誰控制？能否被發行結構機械式推動？`
- 每次更新 M1 equity mNAV 時，必須檢查最新 10-Q/10-K 的淨遞延稅負債或資產。
- 若資料缺失，前端只能標 `需覆核`，不能顯示成精確安全邊際。
